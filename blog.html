Cada semana, un vistazo detrás de cámaras: decisiones, errores y logros.

Sentando las bases del proyecto

El desarrollo de Flowlytics ha comenzado oficialmente, y esta primera semana ha estado dedicada a uno de los momentos más importantes en cualquier proyecto: la preparación. Antes de escribir una sola línea de código funcional, era esencial establecer una base sólida que garantizara escalabilidad, orden y coherencia a largo plazo. Pulsa en "Leer Más" para ver en detalle el trabajo realizado durante estos primeros días.

Estructura inicial del proyecto y repositorios.


Una de las primeras decisiones ha sido crear dos repositorios independientes en GitHub: uno para el frontend y otro para el backend. Esta separación responde a varios motivos prácticos: Permite un control de versiones más limpio. Facilita despliegues independientes según necesidades. Mejora la modularidad del proyecto. Posibilita que, en el futuro, cada parte pueda evolucionar por separado o incluso integrarse con otros servicios. Esta estructura favorece un desarrollo más profesional y flexible, especialmente para proyectos que tienen potencial de crecer o incorporar nuevas tecnologías. Para ilustrar la estructura inicial, se ha elaborado un diagrama en Excalidraw, que muestra cómo interactúan los distintos componentes del proyecto: frontend, backend y base de datos, además de los flujos principales de datos.



Selección del stack tecnológico.


La elección del stack es clave, y Flowlytics necesita fluidez, velocidad y capacidad de adaptación. Finalmente, el stack definido para esta primera versión es: Frontend: React + CSS. Elegido por su capacidad de construir interfaces dinámicas, por la modularidad de sus componentes y por su comunidad madura. Backend: Node.js + Express. Una combinación ligera, eficiente y versátil. Ideal para crear APIs rápidas y para manejar flujos de datos como los que Flowlytics procesará. Base de datos: MongoDB. Su flexibilidad en la gestión de documentos lo convierte en un gran aliado, especialmente para proyectos donde la estructura de datos puede evolucionar. Este conjunto tecnológico permitirá una construcción ágil y escalable, manteniendo un equilibrio entre simplicidad y potencia.



Identidad visual: logo, branding y primer diseño.


Además del aspecto técnico, esta semana también estuvo dedicada a definir la identidad visual del proyecto.Se ha trabajado en: El logo oficial de Flowlytics La paleta cromática La estética base para la interfaz Un primer diseño conceptual del layout. El objetivo ha sido transmitir claridad, análisis y fluidez: los tres pilares que constituirán la personalidad de Flowlytics.



Organización y planificación: Trello.


Para garantizar un desarrollo ágil y bien estructurado, esta semana también se ha avanzado en la organización interna del proyecto. He utilizado Trello como tablero central del proyecto, donde se han definido: Listas de tareas por fases Prioridades Roadmap inicial Fechas estimadas de cada módulo Con Trello, el avance queda perfectamente organizado y medible, lo que facilita mantener el proyecto en movimiento sin perder foco.



Conclusiones de la primera semana.

Esta primera etapa ha estado centrada en la arquitectura inicial y definición de la identidad del proyecto. La estructura ya está lista, el stack está decidido, la imagen de Flowlytics comienza a tomar forma y la planificación y mapa conceptual claros. La próxima semana estará centrada en configurar JWT, conectar el backend con la base de datos y proteger las rutas privadas para comenzar a construir la seguridad y estructura interna del proyecto. El camino apenas empieza, pero Flowlytics ya tiene bases sólidas sobre las que crecer.


Autenticación y seguridad

Esta semana en Flowlytics se ha dado un paso clave: permitir el acceso a la aplicación de manera segura y fluida. La autenticación es la base para cualquier SaaS, y el objetivo era construirla de forma robusta y moderna.

Configuración de JWT.


Se ha implementado JSON Web Tokens (JWT) para manejar sesiones de usuario, lo que permite:


  • Validar la identidad de los usuarios en cada petición
  • Mantener sesiones seguras sin depender de cookies de sesión tradicionales
  • Escalar de manera simple cuando el proyecto crezca


Pantallas de autenticación.


Se crearon las interfaces principales para interactuar con el sistema:


Login → para entrar con email/contraseña o mediante Google

Registro → crear nueva cuenta de usuario

Reset password → recuperar contraseña fácilmente


Además, si el usuario ya está registrado, se implementa login automático, haciendo la experiencia mucho más rápida y cómoda.



Conexión con la base de datos.


Todas las credenciales se almacenan de forma segura en la base de datos MongoDB, utilizando hashing de contraseñas y validaciones estrictas para garantizar integridad y privacidad.



Protección de rutas privadas.


Para que solo los usuarios autenticados accedan a determinadas secciones se ha creado un middleware que verifica el token JWT antes de permitir el acceso. Esto asegura que:


  • Solo usuarios legítimos vean información sensible
  • La app mantiene la seguridad desde el primer acceso


Resultado de la Semana.


Ahora Flowlytics tiene un sistema de login funcional y seguro, permitiendo a los usuarios:


  • Crear cuenta
  • Entrar con email/contraseña o Google
  • Recuperar contraseña
  • Acceder automáticamente si ya están registrados
  • Navegar solo por rutas autorizadas

La próxima semana estará destinada principalmente a construir la estructura visual , integrando todos estos accesos y preparando las bases para mostrar datos reales.

Flowlytics sigue creciendo… y cada semana se hace más funcional y profesional.


Construyendo la estructura base

En esta primera etapa del desarrollo de Flowlytics se ha dado un paso decisivo: la creación de la base visual y estructural que sostendrá todo el sistema en el futuro.

Estas semanas iniciales han estado enfocadas en diseñar y construir los elementos fundamentales que permitirán, más adelante, incorporar funcionalidades reales sin perder coherencia ni estabilidad. El objetivo principal ha sido establecer una plataforma sólida, intuitiva y preparada para crecer de forma orgánica.



La versión inicial ya se encuentra operativa y reúne varios componentes esenciales que marcan el inicio de la identidad visual y funcional de la plataforma:


  • Sidebar funcional: Se ha desarrollado un panel lateral que funcionará como el menú principal de navegación. Su estructura ya está completamente integrada en el flujo de la aplicación, permitiendo un acceso ordenado y consistente a las distintas secciones. Aunque los apartados actuales son provisionales, el diseño ya define cómo se organizará la experiencia del usuario a medida que la plataforma crezca.

  • Navbar superior: Se ha añadido una barra superior que servirá como punto central de interacciones cotidianas dentro de Flowlytics. Este elemento aportará estabilidad y facilitará la incorporación de futuras acciones esenciales, como notificaciones, accesos rápidos o configuraciones del usuario.

  • Vista inicial tras el acceso: La página principal ya está diseñada y preparada para recibir información real. Esta vista, que será la primera impresión al iniciar sesión, busca transmitir claridad, jerarquía visual y un acceso inmediato a los datos más relevantes. Su diseño modular permitirá adaptar la información mostrada conforme la plataforma evolucione.

  • Gráficos placeholder: Se han creado los espacios destinados a las visualizaciones junto con su lógica base. Aunque actualmente muestran contenido ficticio, cumplen una función fundamental: establecer cómo se integrarán los gráficos reales en el futuro. La estructura ya está pensada para incorporar datos dinámicos y ofrecer una experiencia visual rica y comprensible.

  • Tarjetas métricas (KPIs): Los componentes destinados a mostrar indicadores clave ya están maquetados e integrados en el dashboard. Por ahora presentan valores de ejemplo, pero servirán como referencia para la presentación de métricas reales más adelante. Su diseño resalta simplicidad, lectura rápida y relevancia visual.


En conjunto, estos elementos permiten visualizar la dirección que tomará Flowlytics a nivel estético, funcional y estructural. La plataforma comienza a tomar forma y ya ofrece una idea clara del entorno en el que operarán las futuras herramientas y módulos.



De cara a la próxima semana, el trabajo continuará centrado en el diseño y la ampliación de la interfaz. Se explorarán nuevos componentes, se pulirán detalles visuales y se avanzará en la cohesión del sistema, preparando el camino para etapas más complejas. Flowlytics recién comienza, y cada paso está construyendo una experiencia pensada para escalar y evolucionar.


Finalización de la estructura base

Con la primera etapa de Flowlytics llegando a su fin, la plataforma ya cuenta con una estructura visual y funcional consolidada, lista para soportar el desarrollo de nuevas funcionalidades.

En esta fase se ha procedido con un walkthrough completo, permitiendo recorrer por completo la plataforma para asegurar que todos los elementos visuales y funcionales estén alineados y operativos, verificando consistencia, accesibilidad y preparación para la integración de datos reales.



Este recorrido inicial de la plataforma ha permitido definir no solo la estética, sino también la interactividad y el comportamiento de los elementos, asegurando que la experiencia del usuario sea coherente y fluida.



Entre los avances más destacados se incluyen:


  • Interacción de botones y formularios: Todos los botones principales responden a acciones de usuario, y los formularios iniciales permiten la captura de datos de manera estructurada. Esto sienta las bases para futuros flujos de trabajo, validaciones y procesamiento de información dentro de la plataforma.

  • Consistencia visual y diseño coherente:

    • Tamaño, color, borde y tipografía: Todos los elementos se alinean con la guía de estilo de Flowlytics.
    • Espaciado y alineación: Márgenes, padding y disposición de los elementos revisados para una interfaz limpia y ordenada.
    • Transiciones suaves y consistentes con el resto de la interfaz, reforzando la sensación de fluidez.

  • Optimización y mantenimiento:

    • Reutilización: Los botones y formularios se componentizan para mantener consistencia en futuras expansiones.

  • Interactividad y funcionalidad:

    • Click / Acción: Todos los botones disparan la acción correcta



Con esta base visual y funcional ya estable, el siguiente objetivo será avanzar en el backend, comenzando con la creación del CRUD de una de las secciones principales. Este paso permitirá que Flowlytics empiece a trabajar con información real, alimentando el dashboard y convirtiéndolo en una herramienta verdaderamente operativa.


Gestión de Clientes

Tras consolidar la estructura base de Flowlytics, se ha comenzado a implementar funcionalidades críticas para la gestión de clientes, avanzando hacia un entorno completamente operativo y dinámico.

Nuevas funcionalidades implementadas:

  • Tabla de clientes

    Se ha creado una vista completa que permite visualizar de manera clara y organizada toda la información de los clientes, con columnas adaptables y diseño coherente con la interfaz existente.


  • API de clientes

    La plataforma ya permite crear, editar, modificar y eliminar clientes a través de la API, sentando las bases para futuras integraciones y automatizaciones.


  • Vista individual del cliente

    Cada cliente cuenta con su propia ficha detallada, que muestra información clave y facilita la navegación hacia acciones específicas, como editar datos.


  • Estados del cliente

    Se han definido distintos estados para facilitar la identificación rápida de la situación de cada cliente y permitir un seguimiento más eficiente.


  • Buscador

    La tabla de clientes incorpora un buscador dinámico, mejorando la accesibilidad y permitiendo localizar información de manera ágil.



Con la gestión de clientes ya implementada y funcionando, el siguiente objetivo será avanzar en la próxima sección, dando un nuevo paso hacia la operatividad completa de Flowlytics. Este avance permite que la plataforma empiece a operar con datos reales, incorporando de forma progresiva la gestión de facturas de manera estructurada.


Cada nuevo módulo refuerza Flowlytics como un sistema backend coherente y escalable, orientado al seguimiento integral de clientes y su facturación asociada. Este enfoque permite una evolución natural del sistema, facilitando futuras extensiones y manteniendo la coherencia de los datos.



Avance de Sesiones Clave

Tras el periodo navideño, Flowlytics retoma su evolución con avances significativos centrados en reforzar la base funcional de las sesiones clave del proyecto.

Durante estas semanas se ha trabajado en la implementación del CRUD completo de cada una de las sesiones principales, poniendo el foco no solo en la gestión individual de los datos, sino también en la relación y coherencia entre ellos.



Avances realizados:


  • CRUD de sesiones clave: Se ha desarrollado la funcionalidad completa para crear, consultar, actualizar y eliminar cada una de las sesiones fundamentales del sistema, asegurando una gestión consistente y alineada con la arquitectura definida, así como la correcta conexión y dependencia entre las distintas sesiones del proyecto.

  • Relación entre sesiones: Las sesiones ya se encuentran interconectadas, permitiendo que los datos fluyan de forma coherente entre ellas y sentando las bases para una visión global del sistema.

  • Vistas estructuradas: Cada sesión cuenta con vistas claras y organizadas que facilitan la navegación y mejoran la comprensión del flujo de información dentro de la plataforma.

  • Persistencia de datos: La información se gestiona de forma centralizada, permitiendo trabajar con datos reales y manteniendo la integridad del sistema a medida que crece.



Con esta base funcional ya consolidada, el siguiente paso será continuar avanzando en la implementación de métricas y gráficos, una pieza clave para transformar los datos en información práctica.



El objetivo es dotar a Flowlytics de visualizaciones claras y métricas relevantes que permitan analizar la actividad de las sesiones y extraer conclusiones de valor de forma inmediata.



Cada iteración acerca la plataforma a un entorno más analítico y orientado a datos, reforzando a Flowlytics como un sistema escalable, coherente y preparado para crecer de forma ordenada.



Este enfoque progresivo permite asegurar la calidad del desarrollo, manteniendo la consistencia entre módulos y facilitando la incorporación de nuevas funcionalidades en fases posteriores.
